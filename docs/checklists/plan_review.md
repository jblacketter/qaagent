# Plan Review Checklist

Use this checklist when reviewing a phase plan.

## Scope & Feasibility
- [ ] Scope is clearly defined
- [ ] Scope is appropriately sized (completable in reasonable time)
- [ ] Technical approach is feasible
- [ ] Dependencies on other phases are correctly identified
- [ ] No scope creep from original requirements

## Technical Design
- [ ] Architecture decisions are sound
- [ ] File structure is logical and follows conventions
- [ ] Design is appropriate for the use case
- [ ] No over-engineering or unnecessary complexity
- [ ] Patterns used are appropriate for project size

## Success Criteria
- [ ] Each criterion is specific and testable
- [ ] Criteria match the stated scope
- [ ] All major deliverables have criteria
- [ ] Criteria are achievable within the phase

## Risks & Questions
- [ ] Major risks are identified
- [ ] Mitigations are reasonable
- [ ] Open questions are appropriate (not too many)
- [ ] No obvious risks missed

## Alignment with Requirements
- [ ] Phase advances the project goals
- [ ] Design considerations are appropriate
- [ ] Data model supports required functionality
